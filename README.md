# CSI606-2025-02 - Remoto  
## Proposta de Trabalho Final  
### Sistema-de-auto-chamada  

**Discente:** Vitor Angelo dos Santos  

---

## Resumo

Este projeto visa atualizar a forma de realizar a chamada de presença dos alunos nas disciplinas do ICEA. O sistema é inspirado em modelos já utilizados em outras universidades do país. Ele ficará disponível ao aluno e ao professor somente quando a aula for finalizada pelo docente, que permitirá o acesso para confirmação de presença.

---

## Objetivos

• Tornar o sistema de presença em aulas no ICEA mais simples de ser realizado.  
• Facilitar o manuseio para alunos e professores.  
• Integrar tecnologias já utilizadas diariamente, como o MoodleUFOP.  
• Oferecer objetividade e agilidade na confirmação de presença.  
• Facilitar o monitoramento das faltas e presenças.

---

## Especificações

### Para o discente

• A aplicação será integrada ao aplicativo MoodleUFOP, utilizado no cotidiano por alunos e professores.  
• O aluno precisará apenas acessar a disciplina no Moodle e, em uma opção específica, confirmar sua presença na aula correspondente.  
• O professor terá acesso aos alunos que confirmaram presença e, ao final do período, poderá visualizar relatórios com informações consolidadas.  
• Os professores responsáveis pelas aulas também poderão confirmar a participação do aluno.  
• Será possível ao aluno escanear um QR Code disponibilizado pelo professor gerente da sala, em último caso.

### Para o docente

• A aplicação para o professor contará com funcionalidades semelhantes às do aluno e também opções específicas, como relatórios e gráficos com dados da participação dos estudantes nas disciplinas lecionadas.  
• O docente poderá acessar essas informações quando desejar e, ao final do período, terá acesso a um resumo geral, incluindo dados sobre alunos vinculados a programas de auxílio.

---

## 1. Tema

O trabalho final tem como tema o desenvolvimento de um sistema de auto presença para alunos da UFOP, integrado ao aplicativo MoodleUFOP.

---

## 2. Escopo

Este projeto terá as seguintes funcionalidades:

• Criar turmas para uma determinada disciplina.  
• Adicionar/criar alunos para as turmas, assim como professores responsáveis pela disciplina.  
• Alterar/modificar informações sobre a presença do discente e do docente.  
• Visualizar informações em gráficos sobre a presença.  
• Utilizar dados de presença do aluno para outras finalidades (como programas de auxílio universitário).  
• Excluir dados sobre usuários.  
• Definir usuário administrador.  
• Permitir que usuários administrem diferentes páginas, com informações distintas.

---

## 4. Restrições

• Integração real com o MoodleUFOP (será utilizada apenas uma simulação ou protótipo).  
• Integração com bancos de dados reais da UFOP; será utilizado apenas um banco de testes.  
• Disponibilização de aplicativo nativo (Android/iOS); o projeto será demonstrado apenas em ambiente web.  
• Possibilidade de uso offline — o sistema dependerá de conexão à internet.
